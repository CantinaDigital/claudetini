## Guidelines

1. **Be Comprehensive**: Include all common patterns for the detected stack
2. **Be Specific**: Only include patterns relevant to THIS project
3. **Comment Sections**: Group related patterns with comments
4. **Don't Overdo It**: Don't ignore files that should be committed (like package-lock.json)

## What NOT to Ignore

These should typically be committed:
- `package-lock.json` (Node.js) - ensures consistent dependencies
- `Gemfile.lock` (Ruby) - locks gem versions
- `Pipfile.lock` (Python) - locks Python dependencies
- `yarn.lock` - locks Yarn dependencies
- Configuration files (unless they contain secrets)
